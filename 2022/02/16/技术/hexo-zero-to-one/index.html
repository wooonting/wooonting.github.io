hexo from zero to one

安装、初始化环境

install Node.js //安装node.js
npm install -g hexo //安装hexo
hexo init //在一个文件夹下，做hexo初始化

测试使用HEXO

hexo generate //生成静态页面
hexo server //本地启动hexo服务器

关联Hexo与Github

deploy:
type: ‘git’
repository: https://github.com/wooonting/woooting.github.io.git
branch: master

安装插件hexo-deployer-git

npm install hexo-deployer-git –save //使用包管理工具npm安装插件，用于发布Hexo到github上（主要做的工作是：把Public文件夹下的文件复制到.deploy_git下），问题是public下的文件怎么来的？

将hexo目录部署到github上

hexo clean
hexo generate // 生成静态页面
hexo deploy // 部署到github

新建文章

hexo new post “hexo-zero-to-one”
hexo generate // 生成静态页面
hexo deploy // 部署到github

新建草稿

hexo new draft
hexo server –draft // 本地预览草稿

安装主题 hexo-theme-fluid

npm install –save hexo-theme-fluid
然后在博客目录下创建 _config.fluid.yml，将主题的 _config.yml 内容复制进去。

指定主题

修改_config.yml :
theme: fluid # 指定主题
language: zh-CN # 指定语言，会影响主题显示的语言，按需修改

创建[关于页]

hexo new page about

创建成功后，编辑博客目录下 /source/about/index.md，添加 layout 属性。

修改后的文件示例如下：

title: about
date: 2020-02-23 19:20:33
layout: about

更新主题

npm update –save hexo-theme-fluid
