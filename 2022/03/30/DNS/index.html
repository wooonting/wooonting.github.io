DNS

Domain Name Server 域名服务器

DNS 的作用

DNS负责记录IP地址与域名的对应关系。
将域名解析成IP地址，方便主机之间通过IP地址来传输数据包。

正向解析

输入域名，得到IP（网络传输底层需要解析成IP地址）

反向解析

输入IP，得到域名。
没有注册过合法域名的、没有向ISP申请IP反向解析的邮件发送者，一般会被邮件服务器/邮件网关的“反垃圾邮件功能”判断为垃圾邮件。
没有向ISP申请做反向解析的IP地址，通常会被ISP以xxx.dynamic.xxx之类的主机名来管理
邮件网关会根据邮件头中发送方的IP地址，去做反向解析，如果解析到的域名与邮件头中的邮件域不一致，邮件就会被拦截。

比如你用  xxx@name.com 这个邮箱给我的邮箱  123@163.com 发了一封信。163邮件服务器接到这封信会查看这封信的信头文件，这封信的信头文件会显示这封信是由哪个IP地址发出来的。然后根据这个IP地址进行反向解析，如果反向解析到这个IP所对应的域名是name.com 那么就接受这封邮件，如果反向解析发现这个IP没有对应到name.com，那么就拒绝这封邮件。
